## 3. Структура заголовків H1-H6

Поточна структура статті занадто коротка, тому для SEO-версії запропоновано таку ієрархію:

```text
H1: Y2K дизайн у вебі: чому Windows 98 стиль знову працює
  H2: Що таке Y2K дизайн
  H2: Чому Windows 98 стиль повертається
    H3: Ностальгія та впізнаваність
    H3: Простота інтерфейсу
  H2: Як використати ретро UI без шкоди для UX
    H3: Контраст і читабельність
    H3: Швидкість завантаження
  H2: Приклад реалізації в ІПЗ-педії
  H2: Висновок
```

У H2-H3 закладено LSI-ключі: `Windows 98 стиль`, `ретро UI`, `піксельні шрифти`, `ностальгія`, `UX`, `веб-дизайн`.
Структура відповідає informational intent: користувач отримує визначення, причини тренду, практичні правила й приклад.

## 5. Schema.org JSON-LD

Для сторінки статті додано `Article` JSON-LD і `BreadcrumbList` JSON-LD у файлі:

```text
apps/frontend/app/articles/[slug]/page.tsx
```

Скорочений приклад розмітки:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Огляд старих добрих Windows-стилів (Y2K)",
  "description": "Аналіз Y2K стилю, Windows 98 епохи та дизайну початку 2000-х.",
  "author": {
    "@type": "Person",
    "name": "Олексій Іванов",
    "url": "https://frontend-production-0907.up.railway.app/authors/oleksii-ivanov"
  },
  "publisher": {
    "@type": "Organization",
    "name": "ІПЗ-педія"
  },
  "datePublished": "2026-04-15T15:00:00.000Z",
  "dateModified": "2026-04-15T15:00:00.000Z",
  "mainEntityOfPage": "https://frontend-production-0907.up.railway.app/articles/y2k-windows-style-review"
}
```

Після деплою сторінку потрібно повторно перевірити через Google Rich Results Test:

```text
https://search.google.com/test/rich-results
```

## 7. SEO-текст

### Y2K дизайн у вебі: чому Windows 98 стиль знову працює

Y2K дизайн у вебі повернувся не лише через ностальгію. Користувачі втомилися від однакових мінімалістичних сторінок,
де кожен блок виглядає як копія попереднього SaaS-лендінгу. Ретро UI у стилі Windows 98 дає сайту характер: бевельовані
кнопки, контрастні панелі, піксельні акценти й проста структура одразу запам'ятовуються.

#### Що таке Y2K дизайн

Y2K дизайн - це візуальна мова кінця 1990-х і початку 2000-х років. Для неї характерні яскраві кольори, прості вікна,
системні шрифти, декоративні рамки, псевдо-3D кнопки та відчуття раннього інтернету. У сучасному вебі цей стиль
використовують не як копію старих сайтів, а як спосіб створити впізнаваний інтерфейс.

#### Чому Windows 98 стиль повертається

Windows 98 стиль працює, бо він дуже швидко пояснює користувачу ієрархію сторінки. Заголовок вікна виглядає як
заголовок блоку, кнопка схожа на кнопку, а рамка чітко відділяє один елемент від іншого. Такий підхід може бути
корисним для навчального IT-блогу, де важливо не тільки виглядати незвично, а й швидко вести читача до контенту.

#### Як використати ретро UI без шкоди для UX

Головне правило: Y2K дизайн не повинен ламати читабельність. Якщо фон занадто строкатий, текст дрібний, а анімацій
забагато, користувач швидко закриє сторінку. Тому ретро-естетику краще використовувати в контрольованих місцях:
заголовках, кнопках, навігації, картках статей і декоративних деталях.

Для SEO також важливо, щоб сторінка залишалася швидкою. Ретро-графіка не має бути важкою, зображення потрібно
стискати, а текст повинен залишатися доступним у HTML. Саме тому ІПЗ-педія використовує Next.js SSR: Google отримує
готовий HTML, а не порожній контейнер після клієнтського рендерингу.

#### Приклад реалізації в ІПЗ-педії

В ІПЗ-педії Y2K дизайн використано як частину айдентики: темно-сині заголовки, сірі блоки, старі кнопки й візуальна
логіка Windows-інтерфейсів. Це допомагає сайту відрізнятися від типових навчальних блогів і водночас не заважає
індексації, бо контент доступний у серверному HTML.

#### Висновок

Y2K дизайн варто використовувати тоді, коли він підтримує зміст, а не замінює його. Якщо поєднати ретро UI, швидке
завантаження, нормальну структуру заголовків і корисний текст, сторінка буде цікавою для людей і зрозумілою для
пошукових систем. Перегляньте інші матеріали ІПЗ-педії, щоб побачити, як ретро-естетика може працювати в сучасному
IT-блозі.

## 9. Canonical та дублікати

Після доробки canonical генерується через Next.js metadata:

```tsx
alternates: {
  canonical: `/articles/${article.slug}`,
}
```

Очікуваний HTML:

```html
<link rel="canonical" href="https://frontend-production-0907.up.railway.app/articles/y2k-windows-style-review" />
```

Перевірка сценаріїв:

## Контрольні питання

1. **Що таке Helpful Content Update?**  
   Це підхід Google до оцінки того, чи контент створений для людей, а не лише для пошукових систем. Він може впливати не
   тільки на одну сторінку, а й на сприйняття якості всього сайту.

2. **Різниця між `<title>` і `<h1>`.**  
   `<title>` показується у вкладці браузера і сніпеті Google, а `<h1>` є головним заголовком на сторінці. Вони можуть
   відрізнятися, але мають відповідати одному інтенту.

3. **Що таке LCP і чому hero не можна lazy-load?**  
   LCP - час появи найбільшого видимого елемента. Якщо hero-зображення є LCP, `loading="lazy"` затримає його і погіршить
   метрику. Альтернатива - preload або пріоритетне завантаження.

4. **Для чого canonical?**  
   Canonical показує Google основну версію сторінки. Він потрібен для URL з UTM, дубльованих сторінок, сортування,
   фільтрів і схожих версій контенту.

5. **Що таке Schema.org і JSON-LD?**  
   Schema.org - словник структурованих даних. JSON-LD - формат вставки цих даних у HTML. Вони допомагають Google краще
   зрозуміти тип сторінки та можуть впливати на rich results.
